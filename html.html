File Management

Make sure that your files are stored in a logical manner where they are accessible and organized. Stick with one naming convention to avoid confusion. Try to find a way to be both descriptive and brief.

Template

<!DOCTYPE html>
<html lang="en">
      <head>
            <title></title>
            <meta charset="utf-8">
      </head>
      <body>
            <header>
            </header>
            <nav>
            </nav>
            <main>
            </main>
            <footer>
            </footer>
      </body>
</html>

Semantic Elements

The semantic elements are organizational elements within the body element. The header is an element where you would put important information you want to appear at the top of your webpage. It is different from the head, because you don't need to include the title and meta tags. The nav is your navigation bar, which usually consists of links to other pages in your website or even sections below on the same webpage. The main is the main content that you want to appear on your webpage. The footer generally contains contact info, dates updated or published, and any copyright info that you wish to display.

Comments and Contents

To make a comment in HTML, use <!--comment-->. Comments are used to add info to your code without it showing up in the actual webpage.

Symbol Entities are special characters which can be put into a website using special codes. For example, since the less than sign is a reserved character (meaning that typing it could cause a text editor to read it as part of a tag and become confused), it is usually typed using an entity name to prevent any errors in code. The entity name for this symbol is & lt; and the entity name for a nonbreaking space is &nbsp; which looks like this  . You can also create these characters by using their entity number. For example, this star ✩ can be created by using its entity number of &#10025;. Most elements have both an entity name and number.

When creating the main content of your page, there are many helpful tags. Here is a list to get started, shown without the brackets.

  • p: paragraph tag
  • h1: heading 1
  • h2: heading 2
  • h3: heading 3
  • ul: unordered/bulleted list
  • ol: ordered/numbered list
  • li: list element
  • dl: description list, a specific type of list used for terms and their descriptions
  • dt: description list term
  • dd: description list definition
  • br: creates a line break
  • hr: creates a solid line, called a thematic break
  • div: used to create a container element that is helpful for labeling and styling specific parts of the page

Images

It is common to use images in your websites. There are some ways to do this to help organize and simplify the process. For example, make sure your images have clear and descriptive names as opposed to trying to link IMG29931. It is also common to have a folder called Images or images within your project folder, just to keep them organized and out of the way.

To place an image within your webpage, use the img tag as follows

<img src = "images/filename.jpg" alt="a photo of x">

The src attribute tells the tag where to get the file, and the images/ tells it to look in the images folder for a jpg file with name filename. The alt attribute means that if you image doesn't load for any reason, it will instead display the alt text provided. This is used to briefly describe the image (eg the chipotle logo) and is different than the alt text provided alongside images to thouroughly describe them to visually impaired users.

You can also use more attributes such as height and width to describe other features of your image. Additionally, it is commonplace to compress images so that they do not take up an unreasonable amount of space, or to make sure they are the right file type. There are many image compressors online. Finally, there are certain types of files that cannot be used to display an image. Some approved file types include jpg, jpeg, png, gif, and webp. Some inapplicable file types include jfif, html, and .txt or .doc. These will not display an image.

Links

Here is a list of some link types and examples.

hyperlink
text, or sometimes images, that can be clicked to access other content
anchor tag
the HTML tag used to link content on a webpage
eg: <a href="index.html">Home</a>
relative link
hyperlink to another file on your computer
eg: <a href="images/seadragon.jpeg">Photo of Seadragon</a>
image link
a link which displays an image on the screen instead of text
<a href="index.html"><img src="images/file.jpg" alt= "file description"></a>
absolute link
hyperlink to external content via URL
<a href="https://pastawebsite.com">Pasta</a>
email link
a link directly connected to an email address
<a href="mailto: email.address@gmail.com">Email me here!</a>
phone number link
a link directly connected to a phone number
<a href="tel: +1##########">Call me here!</a>
*note: make sure to include country code here!
embedded map
you can actually get a link from google maps that is set up to be embedded in a webpage!
go to google maps -> share -> embed a map -> copy html and you should get an iframe element tag

Note that the source your link takes you to is generally located in the href attribute, or src in the case of images. The text between the opening and closing tags will be what appears on the screen in the case of text links. Also, you can add attributes to your links to specify the manner in which they open. My favorite is target="_blank" which opens the link in a new tab!
